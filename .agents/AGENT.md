# AGENT

## 通用规则

- 代码符合Flake8规范（禁止修改flake8规则）。
- 每次完成编码工作，都需通过`run_unit_tests.sh`运行单元测试；
- 每次完成编码工作，都要通过`run_functional_tests.sh`运行功能测试代码。
- 如果需要生成中间文档，保存到.tmp目录，并在必要的时刻删除。
- 如果要生成临时的验证性脚本，保存到.tmp目录，并在必要的时刻删除。
- 除非得到了用户的明确同意，否则测试代码中严禁出现任何形式的`fallback`。测试代码必须严谨，错就是错，对就是对，没有fallback的空间。
- 修改完成之后给出变更汇总和测试结果汇总。

### 单元测试

- 保存在`tests/unit/`目录；
- 如果修改部分没有对应的单元测试，立刻添加；
- 单元测试是用来测试函数逻辑的，所有调用到真实`litert-lm`接口的地方必须使用`monkypatch`或其他方式绕开。

### 功能测试

- 保存在`tests/functional/`目录；
- 给所有关键函数添加日志，并在进行功能测试时抓取这些日志，只有抓取到了所有关键日志才能证明代码写对了。功能测试脚本应该包含检查日志的逻辑。
- 功能测试必须基于真正的模型和`litert-lm`接口展开。
- 如果对应的功能没有功能测试，立刻添加。

## 用户脚本

你要在根目录保留两个可用的py脚本:
- 实现多轮交互功能：`interactive.py`；
- 实现单次询问功能：`question.py`。
并确保它们一直是好用的。

你要在根目录保留两个可用的shell脚本：
- 运行所有单元测试：`run_unit_tests.sh`；
- 运行所有功能测试：`run_functional_tests.sh`。
并确保它们一直是好用的。
